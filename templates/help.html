Help & Documentation

Everything you need to know about Makix Studio, the available models, and how to get the best results.

Models — which one should I use?

Makix Studio uses Qwen3-TTS models from Alibaba. There are four models installed, each designed for a specific use case. They all run at 12 Hz codec rate (the speed at which audio is reconstructed).

1.7B-CustomVoice

Recommended

The best all-around model. It has a built-in library of high-quality preset voices (Ryan, Ethan, Chelsie, etc.) that you can use immediately — no reference audio needed.

Use when: you want great quality with no setup. Perfect for content creation, demos, podcasts, and most everyday use.

1.7B-VoiceDesign

Creative

Design a custom voice from a text description. You describe the voice you want ("young British female, soft and calm") and the model creates it on the fly.

Use when: you need a unique voice that matches a character or brand, and you don't have a real voice sample to clone from.

1.7B-Base

Voice Clone

Upload a reference audio clip of any voice, and the model clones it. Optionally provide a transcription of the clip to improve accuracy.

Use when: you need to reproduce a specific real person's voice, or recreate a voice from an audio sample you already have.

0.6B-CustomVoice

Lightweight

A smaller, faster version of the CustomVoice model. Same preset voices, but uses about 3× less VRAM and generates audio significantly faster.

Use when: you have limited GPU memory, you need fast generation, or you're doing quick tests and don't need maximum quality.

Context size & character limits

Every model has a maximum amount of audio it can generate in a single run. This is measured in tokens — not characters. The table below shows the limits for each model and what that means in practical terms.

Model Max tokens (model) App limit (MAX_NEW_TOKENS) ~Audio duration ~Characters
0.6B-CustomVoice 8,192 configurable up to ~11:22 min (at 8192) up to ~7,400
1.7B-Base 8,192 configurable up to ~11:22 min (at 8192) up to ~7,400
1.7B-CustomVoice 8,192 configurable up to ~11:22 min (at 8192) up to ~7,400
1.7B-VoiceDesign 8,192 configurable up to ~11:22 min (at 8192) up to ~7,400

The app default is 2,048 tokens — about 2 min 50 sec of audio (~1,800 characters). This is well below the model maximum of 8,192. You can raise it in .env to allow longer outputs.

Character counts are approximate because speech rate varies: fast speakers cover more characters per second than slow, deliberate ones. Languages also differ — Chinese/Japanese are more compact per character than English.

What are tokens, and how do they relate to characters?

When the model generates audio, it doesn't produce raw audio samples directly. It produces audio tokens — small discrete units that a codec then converts into actual sound. This is similar to how language models produce word-piece tokens, which are then assembled into text.

Here's the chain from text to audio:

Text
your script
→
Text tokens
fed as input
→
Audio tokens
model generates these (MAX_NEW_TOKENS)
→
WAV audio
decoded at 12 Hz

At 12 Hz codec rate, the model generates roughly 150 audio tokens per second of speech. So:

MAX_NEW_TOKENS ~Audio duration ~English characters VRAM / speed impact
2,048 (default) ~2:50 min ~1,800 chars Lowest — fast
4,096 ~5:41 min ~3,700 chars Moderate
6,144 ~8:32 min ~5,500 chars Higher
8,192 (max) ~11:22 min ~7,400 chars Highest — slow on low VRAM

For longer content (articles, audiobooks), split your text into paragraphs and generate each one separately. This gives you more control over pacing and avoids hitting token limits.

Configuration via .env

All runtime settings are controlled by a .env file in the root directory of the app. Edit this file and restart the server to apply changes. No code edits needed.

# ── Server ──────────────────────────────────────
HOST=0.0.0.0 # IP to bind (0.0.0.0 = all interfaces)
PORT=8000 # HTTP port

# ── Security ─────────────────────────────────────
SECRET_KEY=change-me # JWT signing key — change in production!
ACCESS_TOKEN_EXPIRE_MINUTES=480 # Session lifetime (480 = 8 hours)
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin # Change this!

# ── Model ────────────────────────────────────────
DEFAULT_MODEL=./Qwen3-TTS-12Hz-1.7B-CustomVoice # Loaded on startup

# ── Generation limits ────────────────────────────
MAX_NEW_TOKENS=2048 # Audio tokens per generation run
# 2048 = ~2:50 min / ~1800 chars (default)
# 4096 = ~5:41 min / ~3700 chars
# 8192 = ~11:22 min / ~7400 chars (model max)
  • 1

    Open the .env file in a text editor.

  • 2

    Change MAX_NEW_TOKENS to your desired value (e.g. 4096 for ~400 character scripts).

  • 3

    Save the file and restart the server (python app.py or your service manager). The new value takes effect immediately on startup.

Don't exceed 8,192 — that is the hard model limit. The generation will either error or silently truncate.

Tips & best practices

Split long texts into paragraphs. Instead of one massive generation, break your script at natural pause points. Each chunk generates faster, and you can re-generate individual parts without re-processing the whole script.

Voice cloning quality depends on your reference audio. For the Base model, use a clean, noise-free clip of 5–15 seconds. Background music or noise will degrade the clone quality significantly.

Use Style Instructions creatively. With CustomVoice, the "Style Instruction" field lets you steer delivery — try values like "excited and fast", "slow and calm", or "whispering".

Low VRAM? Use the 0.6B model. It produces comparable quality to the 1.7B for short texts but uses significantly less GPU memory and generates audio faster.

Language selection matters. Always match the language selector to the language of your text. Setting it to "Auto" works but is less reliable — explicit selection improves pronunciation accuracy.
